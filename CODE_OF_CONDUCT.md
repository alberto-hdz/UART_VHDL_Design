# Code of Conduct

## Our Commitment

We are committed to providing a welcoming and inclusive environment for all contributors, regardless of background, experience level, identity, or perspective. We value respectful collaboration and open communication.

## Expected Behavior

All contributors and participants in this project are expected to:

- **Be respectful** — Treat everyone with courtesy and respect in all interactions
- **Be inclusive** — Welcome diverse perspectives and backgrounds
- **Be collaborative** — Work constructively with others; ask questions, listen to feedback, and offer help
- **Be professional** — Keep discussions focused on technical matters and project goals
- **Be constructive** — Provide feedback in a manner intended to help, not harm
- **Be accountable** — Take responsibility for your words and actions

## Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment, discrimination, or hostile comments based on any protected characteristic
- Intimidation, threats, or aggressive language
- Intentional disruption of project work or discussions
- Personal attacks or insults
- Unwelcome sexual attention or advances
- Plagiarism or misrepresentation of authorship
- Deliberate spreading of misinformation about the project or contributors

## Reporting Violations

If you witness or experience behavior that violates this code of conduct:

1. **Document the incident** — Note the date, time, what happened, and who was involved
2. **Report privately** — Contact the project maintainers at `alberto.hdz1063@gmail.com`
3. **Be specific** — Provide details to help us understand and address the issue
4. **Respect confidentiality** — Reports will be handled with discretion

## Enforcement

Project maintainers will investigate all reports fairly and impartially. Violations may result in:

- A private conversation to clarify expectations
- A request to modify behavior
- Temporary or permanent removal from the project

## Scope

This code of conduct applies to all interactions related to the project, including:

- GitHub issues, pull requests, and discussions
- Commits and code reviews
- Project documentation
- Team communications and meetings
